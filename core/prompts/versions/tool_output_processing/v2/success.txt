Interpretation mode guide:

---

EXTRACTION PRIORITY (always follow this order):

1. PROTOCOL / METADATA (ALWAYS preserve if present)
   - Status codes (HTTP, TCP, application-level)
   - Response headers (Server, Content-Type, Content-Length, Set-Cookie, X-Powered-By, etc.)
   - IPs, domains, ports, protocols
   - Tool execution parameters (flags, scan types)
   - Timestamps, durations, byte counts

2. TOOL-SPECIFIC SIGNALS (identify tool type, extract accordingly)
   - Network scans: open ports, services, versions, OS fingerprints
   - HTTP tools (curl, wget, httpx): status line, response headers, redirects, body indicators
   - Directory brute force (gobuster, dirb, feroxbuster): discovered paths with status codes
   - Vulnerability scanners (nuclei, nikto): findings without interpreting risk
   - Credential tools: extracted hashes, formats, counts
   - Packet capture: protocols, connection counts, anomalies

3. CONTENT / BODY SIGNALS (extract meaningful indicators from response bodies)
   - Page titles, application names, version strings
   - Form fields, login indicators, API endpoints
   - Technology fingerprints (framework signatures, generator meta tags)
   - Error messages that reveal software or configuration

4. COMPRESSION RULES (apply after extraction)
   - Remove boilerplate HTML/JS/CSS — but preserve titles, meta tags, and form structure
   - Aggregate repetitive data: prefer counts/ratios/ranges over full lists
   - Use dominant pattern + exceptions when one outcome clearly dominates
   - Keep representative examples when aggregating
   - Keep findings deduplicated and non-overlapping
   - If sampling mode is `head_middle_tail_sample`, treat input as non-contiguous excerpts (not guaranteed complete coverage)
   - In sampled mode, avoid absolute/exhaustive claims unless explicit totals are present
