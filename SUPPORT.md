# DrowAI Support

DrowAI support is provided on a best-effort basis. The project does not
guarantee response times, resolution times, or compatibility with environments
outside the documented support scope.

Supported versioned releases begin with `v0.1.0`. Eligibility for support is
determined by [SECURITY.md](SECURITY.md) and the documented compatibility scope
of the applicable GitHub Release.

## Support Channels

- Reproducible product defects: open a GitHub issue using the bug form.
- Feature proposals: use the feature request form and describe the user need.
- Installation and usage questions: use GitHub Discussions.
- Suspected vulnerabilities: follow [SECURITY.md](SECURITY.md); never disclose
  sensitive details in a public channel.
- Contribution questions: use the relevant issue or pull request after reading
  [CONTRIBUTING.md](CONTRIBUTING.md).

Search existing documentation, issues, discussions, and release notes before
opening a request.

## Required Information

Support requests should include:

- the DrowAI release version or commit SHA;
- operating system and deployment path;
- relevant Python, Node.js, PostgreSQL, Docker, browser, and runtime versions;
- expected and actual behavior;
- a minimal, numbered reproduction;
- the smallest relevant sanitized log or screenshot.

Remove credentials, tokens, cookies, private targets, personal data, customer
information, and generated task artifacts before submission.

## Support Scope

Maintainers prioritize:

- security-boundary failures;
- data loss or corruption;
- installation and upgrade failures on supported paths;
- regressions in documented behavior;
- complete, reproducible bug reports;
- focused contributions that match project scope.

Maintainers may close requests that are duplicates, cannot be reproduced, lack
requested information, concern unsupported modifications, or fall outside
project scope. The closure reason should identify the missing information,
unsupported condition, or applicable canonical issue.

The following are outside project support scope:

- general cybersecurity consulting;
- target-specific operational assistance;
- help using DrowAI against systems without explicit authorization;
- private one-to-one support;
- guarantees for unreleased commits or unsupported versions.

Security support eligibility is defined exclusively in
[SECURITY.md](SECURITY.md).
